Where the spec stands

Client gaps only make sense against the spec timeline. Three revisions matter right now.

2025-06-18prior

Servers become OAuth resource servers: RFC 9728 discovery, RFC 8707 resource indicators, elicitation, structured tool output.

2025-11-25current

OIDC discovery for auth servers, Client ID Metadata Documents (CIMD) replace DCR as the recommended registration, icons, experimental tasks.

2026-07-28ships in 12 days

Largest revision since launch: stateless protocol (no initialize, no sessions), roots / sampling / logging deprecated, tasks move to an extension, DCR deprecated for CIMD.

The gaps that bite in practice

Per-client quirks that matter more than the checkboxes.

ChatGPT

most restrictive mainstream client
  • Remote-only — no stdio anywhere in the product.
  • Outside web developer mode, connectors and deep research ignore your actual tools: servers must expose exactly two read-only tools named search and fetch in OpenAI's citation schema.
  • Full arbitrary tools require developer mode (Pro/Plus/Business/Enterprise/Edu, web only), with per-call write confirmation.
  • No prompts, sampling, elicitation, or roots on any surface.

Microsoft 365 Copilot

two paths, different rules
  • Copilot Studio: tools + resources only (no prompts); streamable HTTP is the only transport — SSE dropped August 2025; traffic routes through Power Platform connectors, so tenant DLP policies apply; DCR available.
  • Declarative agents: tools only (plus MCP Apps widgets); heavyweight packaging via Agents Toolkit manifests; OAuth limited to clients statically pre-registered in the Teams developer portal — no DCR, no API-key auth.
  • No end-user "paste a URL" flow on either path.

Claude

no sampling on any surface
  • Sampling unsupported everywhere — the one notable Anthropic-side gap.
  • claude.ai web is remote-only; local stdio servers need Claude Desktop or Claude Code.
  • No resource subscriptions; ~150k-character tool-result cap and 300s timeout on hosted surfaces.
  • Claude Code is otherwise the most complete CLI: roots, elicitation, list_changed refresh, and the deepest auth stack surveyed (RFC 9728 + 8414 discovery, DCR, CIMD, pre-registered credentials).

Everyone else

quick hits
  • VS Code is the most spec-complete client, including sampling — which almost nobody else has.
  • Cursor omits sampling from its own capability table.
  • Windsurf caps at 100 tools across all servers.
  • JetBrains documents essentially nothing beyond tools and transports.
  • No client surveyed supports experimental tasks or the OAuth client-credentials extension yet.
Building a server for maximum reach? The intersection of all major clients is: streamable HTTP transport, tools as the only primitive, and OAuth with support for both dynamic and pre-registered clients. Sampling and roots are barely supported anywhere — and both are deprecated in the 2026-07-28 revision — so they're safe to skip. Elicitation and resources are worth shipping, but degrade gracefully where absent.
